# MG솔루션스 클라이언트 개발자 채용 과제

안녕하세요! MG솔루션스 클라이언트 개발자 채용 실무능력 검증 과제 리포지토리입니다.

우리는 3D 센서 기반 자세·체형 분석 솔루션 **Moti-Physio**를 만듭니다.
전 세계에서 **한 달에 4만 명 이상**이 Moti-Physio로 측정하고 있고, 물리치료사·트레이너 같은 전문가들이 매일 쓰는 화면을 우리가 만듭니다.

이 과제는 그 실무의 최소 단위를 그대로 옮겨온 것입니다. 화려함보다 **끝까지 완성하는 것**, 그리고 **"왜 이렇게 만들었는지" 스스로 설명할 수 있는 것**을 봅니다.

---

## 과제 : 팔 재활 분석 프로토타입 — 팔 근육 강화/약화 시각화

### 배경 시나리오

**"팔 재활 치료를 목표로 하는 물리치료사"** 가 환자의 팔 움직임 데이터를 보고,
**어떤 근육이 문제인지 한눈에 파악**하고 싶어 합니다.

이 물리치료사가 쓸 프로그램의 프로토타입을 Unity로 만들어 주세요.

### 왜 이 과제인가요?

과제의 각 단계는 실제 입사 후 하게 될 일과 같은 모양입니다.

| 과제에서 하는 일 | Moti-Physio 실무에서 하는 일 |
| :---- | :---- |
| 센서 스켈레톤 JSON 파싱 → 모션 재생 | 3D 센서 스켈레톤 데이터 수신 → 분석 화면 재생 |
| 근육 기시·정지 기반 기하학적 길이 계산 | 관절 각도·신체 정렬 등 기하학적 측정 항목 구현 |
| 정상 데이터 기준 → 비정상 데이터의 문제 근육 판별 | 정상 범위 대비 편차 분석, 비교 분석 기능 |
| 물리치료사가 이해할 UI | 물리치료사·트레이너가 쓰는 제품 화면 |

## 개발 환경

- **Unity 2022.3 LTS** — 이 리포지토리에 `2022.3.62f1` 기준의 빈 프로젝트가 준비되어 있습니다.
- **팔 근육 3D 에셋**: [Arm Muscles Motion (Unity Asset Store)](https://assetstore.unity.com/packages/3d/characters/arm-muscles-motion-104538)
  - 에셋은 이 리포지토리에 포함되어 있습니다. **과제 목적으로만 사용**해주세요.

## 리포지토리 구성

```
Client_Developer_Test/
└── New Unity Project/     # Unity 2022.3.62f1 빈 프로젝트 (여기서 작업하세요)
Raw_Data/
└── ROM/
    ├── Normal_Data/       # 팔 가동범위가 잘 나오는 모션 데이터 (정상 기준)
    └── Abnormal_Data/     # 팔 가동범위가 안 나오는 모션 데이터 (분석 대상)
```

## 제공 데이터 (`Raw_Data/ROM/`)

각 폴더(`Normal_Data/`, `Abnormal_Data/`)의 공통 구성:

| 파일 | 설명 |
| :---- | :---- |
| `Anterior_Skeleton.json` | 프레임 배열. 각 프레임은 `{ID, NowDataTimestamp, Joints[25]}`, 각 관절은 `{type(관절 인덱스), confidence(신뢰도), REAL(3D 좌표 x,y,z), PROJ(2D 투영 좌표)}` |
| `Anterior_Color.mp4` | 실제 촬영 컬러 영상 (동작 참고용) |
| `Anterior_DepthUser.mp4` | 뎁스/유저 영상 (동작 참고용) |
| `DeepDepth_CaliInfo.json` | 카메라 캘리브레이션 정보 |

> 참고: `confidence`가 낮거나 0인 관절 데이터가 섞여 있습니다. 이를 어떻게 다룰지는 여러분의 몫입니다.

## 요구사항 (필수)

### 1. 모션 재생

- 스켈레톤 JSON을 파싱해, 모션 데이터를 팔 모델에 **IK로 연결**하여 재생하세요.
- 재생 / 일시정지 / 타임라인 탐색이 가능해야 합니다.

### 2. 근육 강화/약화 및 수축 정도 시각화

- 각 근육의 **기시(origin)·정지(insertion)** 를 기하학적인 관점에서 접근하세요.
- 프레임마다 근육의 기시-정지 길이를 계산하여, 기준 대비 **짧아지면 강화(빨강), 길어지면 약화(파랑)** 로 표시하세요.
- **수축 정도**(얼마나 짧아졌는가/길어졌는가)도 함께 표현하세요. (예: 색 강도 그라데이션, 수치 표시 등 — 방식은 자유)

### 3. 정상 vs 비정상 비교 분석

- **정상 데이터(`Normal_Data`)를 기준**으로 삼아, 비정상 데이터(`Abnormal_Data`)에서 **어떤 근육이 문제인지** 표현하세요.
- 기준을 무엇으로 잡았는지(어떤 프레임, 어떤 통계값)는 본인이 정의하고, 그 근거를 README에 설명하세요.

### 4. 물리치료사가 이해할 수 있는 UI

- 개발자가 아니라 **팔 재활 치료를 목표로 하는 물리치료사**가 보는 화면입니다.
- 근육 이름, 상태 요약, 색 범례 등 — 전문가가 환자를 앞에 두고 바로 쓸 수 있는 수준의 화면을 구성하세요.

## 선택 (가산점)

- 팔 가동범위(ROM) 각도 수치화
- 시계열 차트, 결과 요약 리포트 화면
- 성능 최적화 (프로파일링 근거가 있으면 더 좋습니다)

## 제출 방법

1. 이 리포지토리를 **클론**한 뒤, 본인 GitHub 계정에 **private 리포지토리**로 올려 작업하세요.
   - 다른 지원자들이 볼 수 있으므로 **public으로 공개하지 말아주세요.** (fork 대신 private 리포 권장)
2. **작업 과정이 보이는 커밋 이력**을 남겨주세요. 한 번에 몰아서 올리지 마세요.
3. 프로젝트 루트에 **README**를 작성해주세요. 반드시 포함할 것:
   - ① 실행 방법 (Unity 버전, 씬 위치, 조작법)
   - ② 접근 방식 설명 — 특히 **각 근육의 기시·정지를 어디로 정의했고 왜 그렇게 정의했는지**, 강화/약화 판단 기준을 무엇으로 잡았는지
   - ③ 한계와 아쉬운 점
4. (권장) 시연 영상 또는 스크린샷을 함께 올려주세요.
5. 완료되면 본인 리포지토리에 **`MGsolutions-dev` 계정을 collaborator로 초대**하고, **과제를 안내받은 메일에 회신**으로 리포지토리 주소를 보내주세요.
